## 概要

**LocalCSV** は、ブラウザ上で完結する軽量なCSV/TSV/SSVエディタです。

CSV/TSV/SSVデータの処理はすべてブラウザ内で完結し、CSV本文が外部サーバーに送信されることはありません。HTMLファイル1つで動作するため、USBメモリに入れても、自分のサーバーに置いても、どこでもすぐに使い始められます。

> **配布ファイル**: 現在の配布物は `LocalCSV.html` の1ファイルのみです。このファイルにHTML・CSS・JavaScriptがすべて内包されており、単独でCSV編集が可能です。

## 特長

| | 特長 | 説明 |
|---|---|---|
| 🔒 | **CSVデータはローカル完結** | CSVの内容はブラウザ内で処理され、外部サーバーへ送信されません。トラッキングやアクセス解析も一切なし。 |
| 📄 | **ファイル1つで完結** | 持ち運べるHTMLファイル1つだけ。USBメモリに入れても、自分のサーバーに置いても、どこでもすぐに使えます。 |
| ⚡ | **軽量で高速** | 大容量データでもストレスなく動作。最適化されたパース処理により、操作中のラグを徹底排除。 |
| 🌗 | **ダークモード対応** | ライト/ダーク切替をワンクリックで。設定はブラウザに保存され、次回以降も維持されます。 |
| 🇯🇵 | **日本語フレンドリー** | UIは全て日本語。ソート処理も日本語ロケールに対応しています。 |
| 💾 | **自動保存** | 編集データをIndexedDB（利用不可時はlocalStorage）に自動保存し、再読み込み時に自動で作業を再開できます。保存データは最終更新から30日で期限切れ扱いになり、自動的に破棄されます。localStorageも利用できない場合はメモリへ一時フォールバックします（同一タブ中のみ有効）。保存状態はヘッダーにリアルタイム表示されます。 |

## デモ・使い方

### Web版を使う

ランディングページから「ブラウザで利用する」をクリックしてエディタを開きます。

### HTMLをダウンロードして使う

1. `LocalCSV.html` をダウンロード
2. ブラウザでファイルを開く
3. テキストエリアにCSV/TSV/SSVデータをペースト、またはCtrl+Vで貼り付け
4. 編集開始！

### コードをコピーして使う

1. ランディングページの「コードをコピー」をクリック
2. 画面上部中央（固定ナビの下）に表示されるトーストでコピー結果を確認
3. 取得したHTMLソースを任意のエディタに貼り付け
4. `LocalCSV.html` として保存してブラウザで開く

> **補足**: ランディングページの「HTMLをダウンロードする」「コードをコピー」は、通常配信では `LocalCSV.html` を優先取得し、失敗時はLP内に埋め込んだエディタHTMLへフォールバックします。`file://` で開いた環境では埋め込みを直接利用するため、コピー導線が失敗しにくい設計です。

> **ヒント**: ダウンロードしたHTMLファイルはインターネット接続なしでもCSV編集機能とアイコン表示を含めて動作します。

## 機能一覧

### データ入力

- **ペーストゾーン**: 画面中央のエリアをクリック、またはCtrl+Vでデータを貼り付け
- **ファイル選択**: `CSV / TSV / SSV / TXT`（`.csv` / `.tsv` / `.ssv` / `.txt`）をインポートウィザード経由で読み込み
- **区切り文字の自動判別**: カンマ（CSV）、タブ（TSV）、セミコロン（SSV）を自動で検出
- **RFC 4180準拠**: ダブルクォート内のカンマ・改行・エスケープに対応
- **改行コード**: CRLF（`\r\n`）、LF（`\n`）、CR（`\r`）のすべてに対応
- **列数の自動補完**: 行ごとに列数が異なる場合、最大列数に合わせて自動的に空セルを補完
- **空行保持**: 原文の空行を保持してパース

### テーブル編集

- セルをクリックしてインライン編集
- セルの変更は自動保存（フォーカスが外れた時点で確定）
- 列ヘッダーのクリックでコンテキストメニューを表示
- 行番号のクリックでコンテキストメニューを表示
- アイコンのみの操作ボタンにはカスタムツールチップを表示（表示遅延はアプリ側で調整可能）
- 行/列のドラッグハンドルにホバーすると `grab` カーソル、ドラッグ中は `grabbing` カーソルで状態を視覚化
- ドラッグハンドルのクリックではメニューを開かず、ドラッグ操作に専念できる挙動

### 列操作

列ヘッダーのコンテキストメニューから以下の操作が可能です。

| 操作 | 説明 |
|---|---|
| **昇順ソート** | 列を昇順にソート。数値は数値順、テキストは日本語ロケール対応のlocaleCompareで比較 |
| **降順ソート** | 列を降順にソート |
| **列フィルタ** | 値一覧から表示する行を絞り込み |
| **列コピー** | 列全体をクリップボードにコピー（改行区切り） |
| **列を複製** | 選択列の右側に同じ内容の列を挿入 |
| **列を削除** | 確認ダイアログ表示後、列を削除 |
| **列ドラッグ移動** | 高速ドラッグ時でも列順の反映が安定するよう改善 |

### 行操作

行番号のコンテキストメニューから以下の操作が可能です。

| 操作 | 説明 |
|---|---|
| **行コピー** | 行全体をTSV形式でクリップボードにコピー |
| **行を複製** | 選択行の下に同じ内容の行を挿入 |
| **行を削除** | 確認ダイアログ表示後、行を削除 |
| **行ドラッグ移動** | 列フィルタ適用中や高速ドラッグ時でも行順の反映が安定するよう改善 |

### 検索・置換

- **リアルタイムハイライト**: 入力中に一致箇所をハイライト表示（大文字小文字を区別しない）
- **前後ナビゲーション**: 「前へ」「次へ」ボタンで一致箇所を順番に移動
- **置換**: 現在の一致箇所を1つだけ置換、または一括置換
- **マッチ情報**: 「3 / 15 件」のように現在位置と総マッチ数を表示
- **スコープ**: ヘッダー行が有効な場合、ヘッダーを除いたデータ行のみを検索
- **詳細検索**: 大文字小文字の区別、完全一致、正規表現、対象列の指定に対応
- **検索履歴**: データをすべてクリアした場合は、検索欄の候補履歴も削除

### ヘッダー行管理

- ツールバーのトグルスイッチでヘッダー行のON/OFFを切替
- ヘッダーON時: 1行目がヘッダーとして固定表示され、行番号は2行目から開始
- ヘッダーOFF時: 列番号（1, 2, 3...）が列名として表示
- ヘッダー設定は元に戻す・やり直しの対象

### 元に戻す・やり直し

- **Undo/Redo**: 最大20回分の操作履歴を保持
- 対象操作: セル編集、行/列の追加・削除・複製、ソート、置換、ヘッダー切替
- ツールバーのボタン、またはキーボードショートカットで操作
- 履歴が空の場合、ボタンは自動で無効化

### エクスポート

| 操作 | 形式 | 出力先 |
|---|---|---|
| **CSVコピー** | CSV（カンマ区切り） | クリップボード |
| **TSVコピー** | TSV（タブ区切り） | クリップボード |
| **CSVダウンロード** | CSV（UTF-8 BOM付き） | ファイル保存 |
| **TSVダウンロード** | TSV（UTF-8 BOM付き） | ファイル保存 |

- ファイル名の形式: `data_YYYY-MM-DD.csv` / `data_YYYY-MM-DD.tsv`
- 値にカンマ・改行・ダブルクォートを含む場合は適切にクォート処理
- **表計算安全モード（推奨）**: `= + - @` 先頭のセルにアポストロフィ（`'`）を付与し、表計算ソフトでの意図しない式評価を防止
- **エクスポートウィザード**: 文字コード（UTF-8/Shift_JIS/EUC-JP）、BOM有無、改行コード、クォートスタイルを選択可能
- **文字コードセレクト**: ヘッダーの統計エリアから現在の文字コードを確認・変更可能

### 列幅リサイズ

- 列ヘッダーの境界をドラッグして列幅を調整（最小幅: 60px）
- 値やヘッダー文字列より細くしても、列幅指定を優先して省略表示できる
- 設定した列幅はlocalStorageに保存され、次回読み込み時に復元

### テーマ切替

- ヘッダー右上の `light_mode` / `dark_mode` アイコンボタンでライト/ダークモードを切替
- 初回起動時（保存済み設定がない場合）はダークモードで開始
- 選択したテーマはIndexedDB（フォールバック時はlocalStorage）に保存
- アクセントカラーパレット（左端かつデフォルト: `#10b981` / `#2563eb` / `#06b6d4` / `#f59e0b` / `#8b5cf6` / `#ec4899`）

### データ永続化

- 編集中のデータは操作ごとにIndexedDBへ自動保存
- IndexedDBが利用できない場合はlocalStorageへ自動フォールバック（この場合も再読み込み後に再開可能）
- localStorageも利用できない場合はメモリへ自動フォールバック（同一タブ中のみ有効、再読み込み/再起動で消去）
- 再読み込み時は、保存済みデータがあれば自動で読み込み
- 保存データは最終更新から30日で期限切れ扱いになり、自動削除
- 保存対象: データ内容、ヘッダー状態、区切り文字、列幅、文字コード、BOM、改行コード、クォートスタイル
- データをすべてクリアした場合は、保存データ、列幅、検索履歴をまとめて削除
- 再読み込み後の継続可否は保存先に依存（IndexedDB/localStorage: 可能、メモリ: 不可）
- ストレージ容量超過時のエラーハンドリングも実装済み

### 保存状態の可視化

ヘッダーの統計エリアに保存状態バッジをリアルタイム表示します。

| 状態 | 表示 | 意味 |
|---|---|---|
| 保存済み | 🟢 `保存済み` | IndexedDBへの保存が完了 |
| 保存中 | 🟡 `保存中…` | 自動保存を実行中 |
| 未保存 | 🟠 `未保存` | 変更あり、まだ保存されていない |
| 一時保存のみ | 🟠 `一時保存のみ` | IndexedDB/localStorageが利用できないためメモリ一時保存中。再読み込みでは再開不可、閉じる前にエクスポート推奨 |
| 保存不可 | 🔴 `保存不可` | 保存に失敗。手動エクスポートを推奨 |

## CSV/TSV/SSV対応仕様

### パース（読み込み）

- RFC 4180準拠のCSVパーサーを内蔵
- カンマ区切り（CSV）、タブ区切り（TSV）、セミコロン区切り（SSV）の自動判別
- ダブルクォートによるフィールド囲み、エスケープ（`""` → `"`）に対応
- CRLF（`\r\n`）、LF（`\n`）、CR（`\r`）のすべての改行コードをサポート
- フィールド内の改行（クォート内）もパース可能
- 空行を保持してパース（データ行として扱い、行の消失を防止）
- 閉じ忘れクオートを検出し、エラートーストで通知

### エクスポート（書き出し）

- DSV出力形式は `CSV / TSV / SSV / TXT` をサポートし、区切り文字に応じて既定形式を選択
- DSV拡張子は `csv / tsv / ssv / txt` を使用（セミコロン区切りは `.ssv`）
- JSON系出力は `JSON (.json)` と `NDJSON (.ndjson)` をサポート
- 値にカンマ、ダブルクォート、改行のいずれかが含まれる場合、自動的にダブルクォートで囲む
- ダブルクォートのエスケープ（`"` → `""`）を適切に処理
- UTF-8 BOM付きでファイル出力（Excelでの文字化けを防止）
- Shift_JIS / EUC-JP への変換出力にも対応

## 通信と外部リソース

LocalCSVは **CSVデータを外部サーバーに送信しません**。トラッキングやアクセス解析も行いません。

ただし、以下の外部通信が発生する場合があります。

| 項目 | 通信先 | 目的 | オフライン時 |
|---|---|---|---|
| Google Fonts（LPのみ） | `fonts.googleapis.com` / `fonts.gstatic.com` | Webフォント（Inter, Manrope, Noto Sans JP）の取得 | LP表示時のみ。エディタ本体のCSV編集には影響なし |

> **閉域網・イントラネット環境**: CSVデータのローカル処理とアイコン表示は完全に動作します。LPのWebフォントだけはフォールバック表示になる場合があります。

## 開発方法

### セットアップ

```bash
git clone https://github.com/rita-rita24/local-csv.git
cd local-csv
npm install
```

### ビルド

```bash
# TypeScriptのコンパイル（src/main.ts → public/js/main.js）
npm run build

# ウォッチモード（ファイル変更時に自動コンパイル）
npm run watch
```

### ローカルでの確認

`index.html` または `LocalCSV.html` をブラウザで直接開くだけで動作します。ローカルサーバーの起動は不要です。

> **補足**: `LocalCSV.html` はエディタ本体のスタンドアロンファイルです。このファイルへの変更は直接HTML内のCSS/JavaScriptを編集してください。

## 設計思想

### 1. 単一ファイル配布

エディタのHTML・CSS・JavaScriptを1つのHTMLファイルに統合。外部依存なしで動作し、配布・持ち運びが容易。

### 2. プライバシーファースト

CSVデータの外部送信を一切行わず、すべてのデータ処理をブラウザ内で完結。トラッキングやアクセス解析も実装していません。アイコンフォントの取得にのみ外部通信が発生します（[詳細](#通信と外部リソース)）。

### 3. パフォーマンス最適化

効率的なCSVパーサーと最小限のDOM操作により、大容量データでもスムーズに動作。200KB以上のCSVではWeb Workerによる非同期パースに自動切替。

### 4. モダンCSS

CSS Cascade Layers (`@layer`) による明確なスタイル階層、oklch色空間による知覚的に均一な色彩設計、静的トークンを基準に `@supports` で `light-dark()` を重ねる互換性重視のテーマ設計。

### 5. アクセシビリティ

セマンティックなHTML構造、キーボードナビゲーション対応、レスポンシブデザインによるモバイル対応。
保存状態バッジに `role="status"`、トーストに `role="status"/"alert"`、主要モーダルに `role="dialog"` と `aria-modal` を付与し、支援技術で状態変化を把握しやすくしています。
